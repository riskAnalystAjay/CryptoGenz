## How to Contribute

### 1. Fork the Repository
- Click the *Fork* button on the top right of the repository.
- Clone your fork locally:
```bash
git clone https://github.com/your-username/CryptoGenz.git

2. Create a Branch

Always create a new branch for your work:


git checkout -b feature/my-new-feature

3. Make Your Changes

Implement your feature or fix.

Keep your code clean, readable, and commented.

Test your changes in GDB Online or any C# console environment.


4. Commit Your Changes

Write clear and descriptive commit messages:


git add .
git commit -m "Add feature: price history simulation"

5. Push and Open a Pull Request

Push your branch:


git push origin feature/my-new-feature

Open a Pull Request (PR) in the main repository.

Describe your changes and why they are useful.


6. Review

Your PR will be reviewed and suggestions may be provided.

Once approved, your changes will be merged.



---

Guidelines

Keep features console-based and GDB compatible.

Avoid breaking existing functionality.

Document any new feature clearly in the README.md.
